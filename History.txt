version 01.01	48 minutes dev	3 minutes code gen	3 minutes test		53 minutes total

This one isn't really a game. It's to help build French vocabulary. This will
be the first app I develop using Google Gemini.
I got the most frequently used Top 1000 French words from the web. The app 
will be a practice app. It will select a random word, either French or 
English. The user has to type in the translation. 
I wrote up a description for this. I ran into multiple issues. 
This is the URL for that list of 1000 words:
	https://strommeninc.com/1000-most-common-french-words-frequency-vocabulary/
I placed the file with the Top 1000 words in the local folder. The browser 
cannot read text files directly. This is a security issue for the browser.
I changed this to have Gemini write the JavaScript to declare into code.
This works better. There is a defect: it will not go to the next word.
I theorize this is because the UI control is disabled. Yep, that was it.
Gemini provided a fix.
After fixing this, it would skip was the notice if you got it right or wrong.
I asked Gemini to fix that - it did.
This is a functional basis for the app. There will be many changes to follow.

version 01.03	18 minutes dev	0 minutes code gen	4 minutes test		1 hour 38

The next one I thought was a bit tricky. Instead of just right or wrong,
where right had to be spelled perfectly, I wanted a third result: close.
A guess would be close if it had these mistakes:
* Spelled correctly, but missing accent marks
* A letter is missing, but is otherwise spelled correctly
* Two letters next to each other are reversed
Not perfect... but I think close enough. I gave this to Gemini. It provided
test cases for each. All of the tests were accurate. It provided the code.
The first attempt appeared to work!
With this close result, I now want to show a running score total at the
bottom, showing the count of right, close, and wrong. Easy change: first
attempt worked.

To allow more testing, I increased the words in the vocabulary from 20 to 100.
